Every card has a title

This card has a paragraph of text with a link. It also has a second sentance of text with a link.

A card is a simple unit of information. We'll use the < article> tag to represent a card and then style it with CSS.

This Card has lists

Lists are everywhere in web applications. They are used for navagation (a list of links).

  • go here
  • go there
  • go somewhere else

Or as a way to display sets of things.

Glossary

This card collects terms and their definitions and into a Description List Element (dl) for later reference.

tag name
The word inside an HTML tag that defines the element
tag
A tag name and opening and closing delimiters. An closing tag puts a forward slash in front of the tag name.
element
An opening tag, some content, and a closing tag. Some elements dont have closing tags and some elements are self closing.

CSS

Every card has a width

But we never use pixels ("px") to determine the size of a card.

We use measurements that flex, like percents (%), or that relate to the size of the content, like root "em" units (rem), which relate to the size of the element to the text size and thus make the card very readable.

Pro Feedback

August 4th. First class of Primer Pro.

Tonight's class went really well. I thought the change in rooms was a good call. For some reason, the layout, size, and even color of the room seemed to help. The class atmosphere also seemed to be a bit more relaxed. Not sure exactly why. Perhaps less students? Maybe, but I think there was a certain conifdence felt in most people at having "survived" the first Primer Class, lol.

I feel that this class seemed more "interactive" for some reason, and I liked that. The presentation was great as usual.

I'm wondering if it might be more beneficial to take breaks at 7 and 8 only, and for 10-15 minutes, allowing people to get up, take a bathroom break, and sit back down and mess with codepen, look something up, apply something learned on their webdev-notebook, ect. It seems that on the longer breaks this is what I have observed most people do (including myself) and it feels very helpful. The first break 1/2 hour into the sesson has only really distracted me and feels like an unwelcome break in flow. After my commute to class, I'm usually ready to go and focused at the beginning. Breaks later in the class are the ones that really seem to help me keep my focus. I wonder how others feel about this.

Also, for whatever reason, navigating the Github content to follow along with on our computers, is a little time consuming and distracting (such as the Responsive Design PDF last night). As a suggestion, perhaps it should be pulled up on the projection screen, gone over, and then the link posted on Slack during the break or at the end, as most people can see the screen and the focus will be kept on the instructor instead of individual computer screens and keyboard goings-on. I know I have gotten caught up in trying to find the correct Github section and missed some of the oral presentation.

All in all, great class!
